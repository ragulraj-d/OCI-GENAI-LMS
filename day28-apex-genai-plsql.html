01 — Overview

What Are We Building

Oracle APEX has a built-in function that lets your PL/SQL code call any REST API — including OCI Gen AI. You send a question. OCI processes it through a large language model. You get an answer back as JSON. This document covers every line of that call.

02 — Variables (DECLARE block)

Variables and Why Each Exists

PL/SQL — DECLARE 
DECLARE
  l_url      VARCHAR2(500);   -- OCI Gen AI endpoint address
  l_body     CLOB;           -- JSON request we send to OCI
  l_response CLOB;           -- Raw JSON response from OCI
  l_answer   VARCHAR2(32767); -- Extracted AI answer text
  l_status   NUMBER;          -- HTTP status code (200, 401 etc.)
  l_comp_id  VARCHAR2(300);   -- OCI Compartment OCID
VariableTypeWhy This Type
l_urlVARCHAR2(500)URL is a short text string. 500 chars is more than enough.
l_bodyCLOBJSON request can be long (especially with preamble). CLOB holds unlimited text.
l_responseCLOBAI response can be very long. CLOB required — VARCHAR2 would overflow.
l_answerVARCHAR2(32767)Extracted answer text. 32767 is the maximum VARCHAR2 size.
l_statusNUMBERHTTP status codes are numbers: 200, 401, 429, 500.
l_comp_idVARCHAR2(300)OCI Compartment OCID is a long string. 300 chars is safe.
03 — Request Headers

apex_web_service.set_request_headers

PL/SQL 
apex_web_service.set_request_headers(
  p_name_01  => 'Content-Type',
  p_value_01 => 'application/json'
);

What It Does

Sets HTTP headers on the outgoing request. Must be called before make_rest_request.

Why Content-Type: application/json

Tells OCI: "The body I am sending is JSON format." OCI reads this header to know how to parse your request body. Without it, OCI may reject the request or misread the body.

⚠️

Critical for OCI: OCI's request signing algorithm includes the Content-Type header in its signature calculation. If the header is missing from the actual request but was included in the signature, the signature will not match and you will get a 401 error.

Parameters

ParameterValuePurpose
p_name_01'Content-Type'The HTTP header name
p_value_01'application/json'Tells server the body is JSON
p_name_02..10any stringYou can set up to 10 headers
04 — Request Body

Building the JSON Body

PL/SQL — JSON Construction 
l_body :=
  '{'
  || '"compartmentId":"' || l_comp_id || '",'
  || '"servingMode":{"servingType":"ON_DEMAND",'
  || '"modelId":"cohere.command-a-03-2025"},'
  || '"chatRequest":{"apiFormat":"COHERE",'
  || '"message":"' || REPLACE(:P10_QUESTION, '"', '\"') || '",'
  || '"maxTokens":500,'
  || '"temperature":0.5,'
  || '"isStream":false,'
  || '"preambleOverride":"You are an HR assistant."'
  || '}}';

JSON Parameters Explained

ParameterValue UsedWhy
compartmentIdyour compartment OCIDTells OCI which account/folder to bill and use
servingTypeON_DEMANDPay per call. No reserved GPU. Best for development.
modelIdcohere.command-a-03-2025Current active Cohere model on OCI Hyderabad region
apiFormatCOHERETells OCI to use Cohere's request/response format
message:P10_QUESTION valueThe actual user question sent to the AI
maxTokens500Limits response length. Prevents long expensive responses. 500 ≈ 375 words.
temperature0.5Controls creativity. 0=always same, 1=very creative. 0.5 is balanced.
isStreamfalseGet complete answer at once. true would stream word by word (more complex).
preambleOverridesystem prompt textDefines AI's role and behavior before processing user question.
💡

Why REPLACE(:P10_QUESTION, '"', '\"') — If the user types a double quote in their question, it would break the JSON string. REPLACE escapes it with a backslash so JSON remains valid.

05 — The API Call

apex_web_service.make_rest_request

PL/SQL 
l_response := apex_web_service.make_rest_request(
  p_url                  => l_url,
  p_http_method          => 'POST',
  p_body                 => l_body,
  p_credential_static_id => 'OCI_GENAI_CREDENTIAL'
);

What This Function Does

The most important line in the entire code. This built-in APEX function sends an HTTP request to any REST API and returns the response as a CLOB.

Parameters Explained

ParameterValueWhy
p_urll_url (OCI endpoint)The web address to send the request to
p_http_method'POST'We are SENDING data (the question). GET is for reading, POST is for sending.
p_bodyl_body (JSON)The JSON request containing our question and settings
p_credential_static_id'OCI_GENAI_CREDENTIAL'Name of the Web Credential storing OCI API keys. APEX uses this to sign the request.

The OCI Endpoint URL

URL Structure 
https://inference.generativeai.ap-hyderabad-1.oci.oraclecloud.com/20231130/actions/chat
                                 ↑ region                              ↑ API version  ↑ action
✅

Why p_credential_static_id matters: This is how APEX authenticates with OCI. The Web Credential contains your User OCID, Tenancy OCID, Fingerprint, and Private Key. APEX uses these to sign every request using RSA — so OCI knows the call is authorized.

apex_web_service.g_status_code

PL/SQL 
l_status := apex_web_service.g_status_code;

What It Does

After make_rest_request runs, this global variable holds the HTTP response code. Read it immediately after the call.

07 — Parse JSON Response

apex_json.parse + apex_json.get_varchar2

PL/SQL — JSON Parsing 
-- Step 1: Parse the raw JSON string into memory
apex_json.parse(l_response);

-- Step 2: Navigate the JSON tree and extract value
l_answer := apex_json.get_varchar2(
  p_path => 'chatResponse.text'
);

The JSON Response Structure from OCI

JSON — OCI Response 
{
  "chatResponse": {
    "text": "Annual leave entitlement is 18 days per year...",
    "finishReason": "COMPLETE",
    "chatHistory": []
  },
  "modelId": "cohere.command-a-03-2025"
}

How the Path Works

'chatResponse.text' means: go inside the chatResponse object, then get the text value. The dot is the path separator — like navigating folders on a computer.

FunctionPurposeWhen to Use
apex_json.parse(clob)Loads JSON string into APEX memory for queryingAlways first — before any get_ call
apex_json.get_varchar2(path)Returns a text value from parsed JSONFor string values like the AI answer
apex_json.get_number(path)Returns a number from parsed JSONFor numeric values like token counts
apex_json.get_boolean(path)Returns true/false from parsed JSONFor boolean flags
08 — Error Handling

IF / EXCEPTION Block

PL/SQL — Error Handling 
IF l_status = 200 THEN
  -- Success path: parse JSON and display answer
  apex_json.parse(l_response);
  l_answer := apex_json.get_varchar2(p_path => 'chatResponse.text');
  :P10_ANSWER := '<div>' || l_answer || '</div>';

ELSIF l_status = 401 THEN
  -- Authentication failed
  :P10_ANSWER := 'Authentication Error. Check Web Credential.';

ELSE
  -- All other errors — show status and raw response
  :P10_ANSWER := 'Error HTTP ' || l_status || ': ' || SUBSTR(l_response,1,500);

END IF;

EXCEPTION
  WHEN OTHERS THEN
    -- Catches any unexpected PL/SQL or network error
    :P10_ANSWER := 'System Error: ' || SQLERRM;

Why EXCEPTION WHEN OTHERS

If anything unexpected happens — network timeout, ORA error, NULL variable — SQLERRM captures the Oracle error message. Without this block the page would show a generic APEX error with no useful information.

09 — Complete Working Example

HR Assistant — Full PL/SQL Code

This is the complete, production-ready code used in the HR Leave Portal. Every line explained above is combined here.

PL/SQL — Complete HR Assistant Process 
DECLARE
  l_url      VARCHAR2(500);
  l_body     CLOB;
  l_response CLOB;
  l_answer   VARCHAR2(32767);
  l_status   NUMBER;
  l_comp_id  VARCHAR2(300);
BEGIN

  -- ① Compartment OCID — identifies your OCI account area
  l_comp_id :=
    'ocid1.compartment.oc1..aaaaaaaawsdiuqgqqkxt7wav...';

  -- ② OCI Gen AI endpoint for Hyderabad region
  l_url :=
    'https://inference.generativeai.ap-hyderabad-1.oci.oraclecloud.com/20231130/actions/chat';

  -- ③ Build the JSON request body
  l_body :=
    '{"compartmentId":"' || l_comp_id || '",'
    || '"servingMode":{"servingType":"ON_DEMAND",'
    || '"modelId":"cohere.command-a-03-2025"},'
    || '"chatRequest":{"apiFormat":"COHERE",'
    || '"message":"' || REPLACE(:P10_QUESTION, '"', '\"') || '",'
    || '"maxTokens":500,'        -- max words in response
    || '"temperature":0.5,'      -- 0=factual, 1=creative
    || '"isStream":false,'        -- get full response at once
    || '"preambleOverride":"You are an HR assistant for LTIMindtree. Answer only HR questions."'
    || '}}';

  -- ④ Set Content-Type header BEFORE calling OCI
  --    Required for OCI request signature to match
  apex_web_service.set_request_headers(
    p_name_01  => 'Content-Type',
    p_value_01 => 'application/json'
  );

  -- ⑤ Call OCI Gen AI — this is the actual API call
  l_response := apex_web_service.make_rest_request(
    p_url                  => l_url,
    p_http_method          => 'POST',
    p_body                 => l_body,
    p_credential_static_id => 'OCI_GENAI_CREDENTIAL'
  );

  -- ⑥ Read HTTP status code
  l_status := apex_web_service.g_status_code;

  -- ⑦ Handle response based on status
  IF l_status = 200 THEN

    -- Parse JSON and extract the AI's answer text
    apex_json.parse(l_response);
    l_answer := apex_json.get_varchar2(
      p_path => 'chatResponse.text'
    );

    -- Display with HTML styling
    :P10_ANSWER :=
      '<div style="background:#f0f7ff;padding:20px;'
      || 'border-left:4px solid #0066cc;border-radius:8px;">'
      || '<b>AI HR Assistant</b><br/><br/>'
      || REPLACE(l_answer, CHR(10), '<br/>')
      || '</div>';

    -- Clear question box after showing answer
    :P10_QUESTION := NULL;

  ELSIF l_status = 401 THEN
    :P10_ANSWER := 'Authentication Error 401. Check Web Credential.';

  ELSE
    :P10_ANSWER := 'Error HTTP ' || l_status
      || ' — ' || SUBSTR(l_response, 1, 500);

  END IF;

EXCEPTION
  WHEN OTHERS THEN
    :P10_ANSWER := 'System Error: ' || SQLERRM;

END;

Line-by-Line Reference

Line / SymbolExplanation
:P10_QUESTIONColon prefix = APEX page item. Gets the value the user typed in the question box.
REPLACE(...,'"','\"')Escapes double quotes in user input so JSON stays valid.
CHR(10)Newline character. AI responses contain these. Replaced with <br/> so HTML shows line breaks.
SUBSTR(l_response,1,500)Shows first 500 characters of error response for debugging.
SQLERRMBuilt-in Oracle function returning the current error message.
:= NULLClears the page item value. Used to clear question box after answer shows.
